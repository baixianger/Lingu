# Why Lingu Kills Every Other Translator

## The Problem With Every Translation App Today

Open Google Translate. Pick a source language. Pick a target language. Type. Read. Now switch direction — click, click, retype. Need a third language? Open another tab. Copy-paste between them. Repeat forever.

DeepL? Same flow. Just prettier.

Every single translation tool forces you into the **A → B pipeline**. One source, one target, one direction. That's a workflow from 2005.

## Lingu: All Languages, All Directions, Zero Friction

Lingu sits in your menu bar. Click it. You see 2–3 language panels side by side. **Type in any one of them** — the others translate instantly.

That's it. That's the entire UX.

### What Makes This Different

**1. Hub-and-Spoke, Not Pipeline**

Other tools: English → Japanese. Want Japanese → English? Swap. Want to add Chinese? Open a new session.

Lingu: Type anywhere. Everything else updates. English panel is the source? Fine. Switch to the Japanese panel and start typing there instead — now English and Chinese update. No mode switching. No "swap" button. The source is wherever you're typing.

**2. Always One Click Away**

- Google Translate: Open browser → navigate to URL → wait for page load → click text field
- DeepL: Open app → wait for window → select languages → click text field
- Lingu: Click menu bar icon → type

Menu bar apps have **zero window management**. No Cmd+Tab hunting. No minimized windows lost behind 47 browser tabs. Click the icon, it's there. Click elsewhere, it's gone.

**3. No Language Switching Tax**

Working with 3 languages simultaneously is Lingu's default state, not a power-user feature you have to set up each time. Configure once in Settings, use forever.

Translators, language learners, international teams — anyone who works across multiple languages doesn't think in A → B. They think in parallel. Lingu matches that mental model.

**4. Clipboard-Aware**

Copy text anywhere on your Mac. Click Lingu. It's already there, translated. No paste step.

**5. Instant, Not Ceremonial**

300ms debounce. Type, pause, translated. No "Translate" button. No Enter key. No waiting for you to declare you're done typing. It just works.

### The Speed Comparison

| Action | Google Translate | DeepL App | Lingu |
|--------|-----------------|-----------|-------|
| Get to text input | ~3s (browser) | ~2s (app switch) | <0.5s (menu bar click) |
| Translate to 1 language | Type + wait | Type + wait | Type + 300ms |
| Translate to 2 languages | Repeat everything | Repeat everything | Already done |
| Switch direction | Click swap + retype | Click swap + retype | Just type in other panel |
| Quick lookup while working | Context switch | Context switch | No context switch |

### Who This Is For

- **Developers** reading docs in multiple languages
- **Translators** comparing outputs across language pairs
- **Language learners** seeing translations side-by-side in real time
- **International teams** drafting messages for multilingual channels
- **Anyone** who's tired of the translate-copy-paste-switch-translate dance

## The Philosophy

Translation is a **utility**, not a destination. You shouldn't have to "go to" your translator. It should be ambient — always available, never in the way, fast enough that you forget it's a separate tool.

Lingu treats translation the way Spotlight treats search: always one gesture away, shows results as you type, disappears when you're done.

That's why it kills everything else.
